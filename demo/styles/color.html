If you use colors from this palette in your project, only use the variable names provided, NOT hex codes. The LESS variables for colors ship by default with the Framework and are available at the project level. Using them ensures consistency across Encore as colors shift or change over time, and will save you from having to revise hard-coded hex codes in the future. Hex codes are provided here for reference only.

We have a reference to the old 1.0 color palette for your convenience, but note that the 1.0 palette is slated for deprecation and shouldn't be used. Any styles that use the 1.0 palette should be updated.

Color Usage

Color is a tool that helps communicate with Encore users. Certain colors in Encore are reserved for particular actions of events.

Green - Create/Add
700
@green-700 #2E7D32
  • Used for links and icons that deal explicitly with create and add actions.
  • Should not be used to communicate "active" or "success" (since those are states, not actions a user can take)
Green - Success/Active
100
@green-100 #C8E6C9
  • Used for statuses that communicate success in notifications.
  • Used for displaying active status in tables and status bubbles.
Blue - Standard Link
700
@blue-700 #1976D2
  • Used for links that direct the user to a new page or area.
Blue - Informative
100
@blue-100 #BBDEFB
  • Used for statuses that are meant to be informative, but are not associated with success, failure, warning, or error.
Gray - Disabled
300
@gray-300 #E0E0E0
  • Used to communicate a button or element is disabled.
Gray - Disabled
600
@gray-600 #9E9E9E
  • Used to communicate that an icon or link is disabled.
Gray - Cancel
700
@gray-700 #757575
  • Used to communicate that an icon, button, or link is associated with a cancel action.
  • For plain text, this gray communicates that something is less important so that darker, primary text content stands out more.
Orange - Warning/Critical Change
700
@orange-700 #D14904
  • Used to mean "warning" where the user's attention or confirmation is needed before taking an additional action.
  • Used for links and icons that deal with editing, changing, or updating something. In some cases this may cause an irreversible change to that object.
Red - Error/Destructive
700
@red-700 #D32F2F
  • Used to mean "error", either as part of a response or status message.
  • Used for link colors and icons associated with destructive/destroy actions. Irreversible changes (like rebuilding a server) are not considered destructive actions.
