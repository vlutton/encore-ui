Directives

Labels and Inputs with rxFormItem

  • The directive rxFormItem creates a field row wrapped in a label. Used for fields which include a single input/field.
  • This will also apply to the textarea tag.
  • You would most likely use this angular.js directive directly, otherwise any input field can be placed inside an element with a class of field-input to apply styling.

Compressed Layout

  • Wrapping form items in a container of class `.rx-form.layout-compressed` will affect the form elements as shown.
  • ENCORE-SPECIFIC: These particular input stylings are used specifically within TicketQueues.

Fieldsets with rxFormFieldset

  • The rxFormFieldset directive creates a field row wrapped in a fieldset. Used for fields which include multiple inputs (e.g. rxFormOptionTable).

Selects with rxFormItem

  • The rxFormItem directive replaces the default UI of select fields and adds a custom border/arrow.
  • You would most likely use this angular.js directive directly, if you are not using angular you could wrap the elements with .field-select.

Checkboxes and radios with rxFormOptionTable

  • rxFormOptionTable is an angularjs-based sub-directive of rxForm which displays checkboxes or radio buttons on passing in an object.
  • ENCORE-SPECIFIC: For radio buttons, while the majority of radio buttons have been used in the context of a data table, there is at least one instance in Encore where they are used as part of the label/input form combo used within rxFormItem: asking the user to do a hard/soft reboot while in the server detail page.

Form Validation

  • Note: Because Encore is built in angular, we leverage the ng-required directive, and as a result the Encore-UI library does have a custom CSS if an element has both ng-invalid and ng-dirty classes, which are dynamically added.
  • Designing an Encore-UI specific design pattern for form validation is on our roadmap.

Design Patterns within Encore

  • Forms can be used on their own page. You can see this in Encore when you create a new object such as a Cloud Server or a Database under Encore Cloud.
  • Forms are also used within modals. This can be see when modifying content that requires form fields such as actions performed on a server detail page.
  • See the section on links and buttons and the sample Create Form for details about how we style and color buttons, but in a nutshell: submit buttons should use the green submit class unless you're deleting or formatting, which should be the negative class. If you need to use a button in your field for auxillary purposes, make it blue.
  • Wells can be used to create additional context for the form.

UI Roadmap / Possible Future-work

  • Fleshing out a design pattern for edit states. Up until now, the editing of content has been relegated to using modals to edit individual line items. As a result, different products have handled the concept of an edit state differently. There should be conformity for this, but we have not designed a user pattern yet.
  • HEADS UP: At this time, Encore only uses stacked forms, and do not offer a design pattern for inline forms. This may change later.
