Basics

You can style most of your buttons using this .button class. Exceptions can include input type="button" or rxButton for dynamic loading indicators.

Button Sizing

Buttons are sized using xl, lg, sm and xs classes. Not adding the additional class defaults the size to medium.

Button Types & Using Color

ENCORE-SPECIFIC: in the context of EncoreUI, when is a button blue and when is it green?

  • If the form you are submitting results in CREATING or ADDING something, use green.
  • If the form you are submitting is within a modal, use green.
  • If the form you are submitting results in EDITING something or something auxiliary like getting metadata from a different source, keep it blue.
  • If you are providing a button for the user to continue through a series of modals (with no submit), use blue.
  • If you are providing a button for the user to exit a series of modals where no other action is required by the user, use the "Finish & Close" button with a blue outline.

Button with Animation

You can use the spinner style with buttons. Different spinner styles are used when content is loading or being fetched (see Using Colors and Icons With Links demo).

When a user takes an action with results that can't be immediately displayed, a spinner is shown on a disabled button state until the call completes or the results of the user's action can be displayed in the UI. Text for your buttons with spinners should describe the action taken using "-ing" verbs, so the user understands why the button appearance has changed. The disabled state ensures the user can't submit the same data multiple times while the initial action completes.

Button Groups

This control should NOT be used as a replacement for radio inputs. Instead, it should be used to indicate state on a page. For example, the legend of a time plot may allow the user to select a range of week, month, or year. In this situation, there is no form being submitted, but there is an indication of the graph's state that can be interacted with. Conversely, radio inputs are the correct way to implement a form input where one of a few options can be selected.

Also, this type of control should only be used with at most four segments.
