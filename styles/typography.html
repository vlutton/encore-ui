Headings/Title Styles

  • Headings are sized according to their heading level (H1 larger than H2 larger than H3, etc.). You can use H3 for most sub-headers.
  • Warning: The .title, .xl, .lg, .sm, and .xs heading classes are deprecated and will be removed in a future release of the framework.

Descriptions / Metadata

  • To display a list of terms with their associated descriptions, use rxMetadata. This is used throughout Encore to show metadata, especially on an individual object such as a server or a CBS volume.
  • ENCORE-SPECIFIC: If you have between 5-15 definitions over 3 columns, rxMetadata is an ideal solution. You can see good examples on server details, or image details pages. However, take note that server details and image details pages are part of Encore Cloud UI. They are not part of the EncoreUI framework.

When There is No Data to Display

  • In all displays of data, you should use "N/A" where no relevant details are available.
  • "Empty", "Null", or blank spaces should be avoided as they may be considered valid inputs.
  • A separate style exists to communicate when data isn't present due to permissions , or when an entire table is empty. see Null Patterns section of Tables style guide.

Lists

  • Lists by default do not have styles. You can add bullets/decimals to lists with the .list class. It is also possible to build non-semantic lists using the additional .item class.
  • ENCORE-SPECIFIC: Use this in conjunction with icons, colors and helper classes such as .flush-right to build out action menus. Look at the detail pages in Cloud Servers and view-source for additional context.
